設計理念

這次製作「兒童學習樂園」網頁，目標是打造一個簡單、好玩、看得懂的學習工具，讓孩子不需要大人指導也能自己操作。

選擇單一 HTML 檔案的作法，是希望降低使用門檻 — 任何人只要拿到檔案，雙擊就能在瀏覽器打開，不需要伺服器、不需要網路、不需要安裝任何東西。這對家長或老師來說是最方便的方式。

遊戲設計的取捨

原本想設計更多種類的遊戲，但最後聚焦在三個核心模式：

  • 數數字：最直觀的數量對應，用 Emoji 取代實體圖片，省去繪圖資源
  • 加法：結合圖像與算式，幫助孩子建立「數量 + 數量 = 總數」的具體概念
  • 認動物：從圖像到文字的配對，讓孩子在遊戲中自然認識國字

三個模式共用同一套架構（10 題一輪、計分、回饋），維護起來很乾淨，擴充新題型也很容易。

技術細節

  • CSS 動畫：答錯的搖晃效果、過關的彩色紙花，都是純 CSS 與少量 JavaScript 實現，沒有引入任何外部函式庫
  • 響應式設計：使用 grid 搭配 auto-fit，選項按鈕在手機上會自動換行，不需另外寫 media query
  • 防重複點擊：作答後立即鎖定按鈕，避免連續點擊造成分數異常

可以改進的方向

  • 加入音效（答對/答錯提示音），但目前怕增加檔案複雜度所以先省略
  • 支援更多題型，例如減法、形狀配對、注音符號
  • 記錄歷史成績，讓孩子看到自己的進步
  • 加入計時模式增加挑戰性

小結

這次的開發過程最大的體會是：兒童產品的設計關鍵不在於功能多，而在於每一個環節都讓孩子感到安全與鼓勵。錯誤時不扣分、完成時有慶祝、用顏色和動畫給予即時回饋 — 這些細節比華麗的特效更重要。
